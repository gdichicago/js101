Control Flow

if statements

The if Statement

Use if to tell JS which statements to execute, based on a condition.

What if looks like:

if (condition) {
  // statements to execute
}

How if Works

if statements execute only if your condition is true. Example:

var x = 5;

if (x > 0) {
  console.log('x is a positive number!');
}

Operators

Comparison Operators

Use these operators to compare two values for equality, inequality, or difference.

Comparison Operators

  • Compares two operands (items) and returns true or false
  • Operands can be numerial, string, logical, or object values
  • If the operands aren't the same type, JavaScript tries to convert before comparing, except for...
  • Strict comparisons don't attempt conversion

Equality

var myAge = 28;
myAge == 28;

Important!
Don't confuse assignment (=) with equality comparisons (==)!

Strict equality

Does not convert objects!

var myAge = 28;
myAge === 28;

What happens when we do this?

var age = '28';
age === 28;

Inequality

var myAge = 28;
myAge != 28;

Strict inequality

var myAge = 28;
myAge !== 28;

Greater than

var myAge = 28;
myAge > 28;

Less than

var myAge = 28;
myAge < 28;

Greater than or equal

var myAge = 28;
myAge >= 28;

Less than or equal

var myAge = 28;
myAge <= 28;

Operators

Logical

And, or, not.

Logical Operators

  • Typically used with comparison operators
  • Compare two operands and returns true or false
  • Allow you to test several conditions at once!
  • Always read left to right
  • Only continue testing if needed

&&

Returns true if: both statements are true.

var posNum = 4;
var negNum = -2;

True expressions:

posNum > 0 && negNum < 0
4 > 0 && -2 < 0

False expressions:

posNum > 0 && negNum > 0
4 > 0 && -2 > 0

||

Returns true if: either statement is true.

var posNum = 4;
var negNum = -2;

True expressions:

posNum > 0 || negNum > 0
4 > 0 || -2 > 0

False expressions:

posNum > 0 || negNum < 0
4 > 0 || -2 < 0

!

Returns true if: the statement is not true.

var posNum = 4;
var negNum = -2;

True expressions:

!(posNum === negNum)
!(posNum < 0)

False expressions:

!(posNum !== negNum)
!(posNum > 0)

Using mutiple operators

When combining together multiple conditions, use parentheses to group:

var myAge = 28;
if ((myAge >= 0 && myAge < 3)  || myAge > 90) {
  console.log('You\'re not quite in your peak.');
}

Logical operators are typically used with Boolean (logical) values; when they are, they return a Boolean value. However, the && and || operators actually return the value of one of the specified operands, so if these operators are used with non-Boolean values, they may return a non-Boolean value. The logical operators are described on the slides.

It is important to remember that in both scenarios conditions are tested left to right, and only continue testing if needed.

Operators

An important lesson: truthy and falsey

Truthy

If you don't use a comparison or logical operator, JavaScript tries to figure out if the value is "truth-y".

var dogsRule = true;
if (dogsRule) {
  console.log("Yay dogs!");
}

Falsey

Values that are "false-y": false, the empty string (""), the number 0, undefined, null.

var name = '';
if (name) {
  console.log("Hello, " + name);
}

var points = 0;
if (points) {
  console.log("You have " + points + " points.");
}

var firstName;
if (firstName) {
  console.log("Your name is " + firstName);
}

Short-Circuit Evaluation

How JavaScript saves us time!

Short-Circuit Evaluation

JavaScript evaluates all logical operators from left to right. It stops evaluating as soon as it knows the answer.

When it stops depends on the operator you're using.

Short-Circuit Evaluation: &&

Stops as soon as it hits a false expression because it requires all things to be true.

var name = "";
var age = 28;
if (name && age) {
  console.log("Hello!);
}

Because name is false, it won't ever get to age.

Short-circuit evaluation: ||

Stops as soon as it hits a true expression because it only needs one thing to be true.

var name = "";
var age = 28;
if (name || age) {
  console.log("Hello!);
}

name is false, but age is true, so this goes the whole way through.

If we change this around to:

if (age || name) {
  console.log("Hello!);
}

age is true, so it stops evaluating.

Control Flow

if, else if, and else statements

if/else statements

Use else to give JavaScript an alternative statement. else runs when the if statement doesn't run.

var age = 28;
if (age >= 16) {
  console.log("Yay, you can drive!);
} else {
  console.log("Sorry, but you have " + (16 - age) + " years until you can drive.");
}

What can we change age to if we wanted to make this hit else instead of if?

if/else if/else

If you want multiple checks, you can use else if to run through them all.

var age = 20;
if (age >= 35) {
  console.log("You can vote and hold any place in government!");
} else if (age >= 25) {
  console.log("You can vote and run for Senate!");
} else if (age >= 18) {
  console.log("You can vote!");
} else {
  console.log("You have no voice in government!");
}

The order of your conditions matters. Very important to remember! JavaScript will pick the first match, not the best match.

Loops

while loop

while loop

The while loop tells JavaScript to repeat statements until a condition is true. A while statement looks like this:

while (expression) {
// statements to repeat
}

while loop

var x = 0;
while (x < 5) {
  console.log(x);
  x = x + 1;
}

Be very careful! It's easy to accidentally cause an infinite loop where your code never stops running.

Loops

for loop

for loop

The for loop is another way of repeating statements, more specialized than while. There are more pieces:

for (initialize; condition; update) {
// statements to repeat
}

for loop

What it looks like with real values:

for (var i = 0; i < 5; i = i + 1) {
  console.log(i);
}

We can use i, j, k, etc. It's harder to do an infinite loop with for, but it's still possible (like if you make a typo).

To save space, we can use this for the update part: i++. It is shorthand for: i = i + 1. Most people use this!

Loops

Breaking out of loops with break.

break statement

To prematurely exit a loop, you can use the break statement:

for (var current = 100; current < 200; current++) {
  console.log("Testing " + current);
  if (current % 7 == 0) {
    console.log("Found it! " + current);
    break;
  }
}

Arrays

A data type containing an ordered list of values.

Arrays

Arrays are a data type that hold an ordered list of values. These values can be any type. Arrays look like this:

var arrayName = [element0, element1, element2];

We can make an array of our favorite things:

var favoriteThings = ["kittens", "brown paper packages", "ribbons"]

And find out the length:

console.log(favoriteThings.length);

Accessing things in arrays

You can access things in an array using bracket notation and the position. The first item in your array is at position 0.

var rainbowColors = ['Red', 'Orange', 'Yellow', 'Green', 'Blue', 'Indigo', 'Violet'];
var firstColor = rainbowColors[0];
var lastColor = rainbowColors[6];

What is firstColor? What is lastColor?

Changing items in arrays

You can also use bracket notation to change items in the array.

var myFavoriteThings = ['Broccoli', 60481, 'Love Actually'];
myFavoriteThings[0] = 'Celery Root';

Now when we call favorieThings[0], what do we get?

Adding items to arrays

If you know how many items are in an array, you can use bracket notation to add things to the end.

var myFavoriteThings = ['Broccoli', 60481, 'Love Actually'];
myFavoriteThings[4] = 'Sunshine';

If you don't know how many items you have, there's a handy method for this called push(). It looks like this:

myFavoriteThings.push("Dancing");
  console.log(myFavoriteThings);

What's in our array now?

Iterating through arrays

Use a for loop to easily process each item in an array:

var rainbowColors = ["red", "orange", "yellow", "green", "blue", "indigo", "violet"];

for (var i = 0; i < rainbowColors.length; i++) {
  console.log(rainbowColors[i]);
}
